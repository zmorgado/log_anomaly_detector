# Log Anomaly Detector

A hybrid two-stage system for detecting and classifying network security threats, built as the final project of the *Máster en Inteligencia Artificial* (CEI Sevilla).

It combines an **unsupervised LSTM Autoencoder** that flags anomalous network flows with no prior knowledge of the attack, and a **supervised multiclass XGBoost** classifier that identifies the specific threat type for the flows marked as anomalous.

## Motivation

Security Operations Center (SOC) teams are flooded with security alerts, which forces highly skilled staff to spend time triaging incidents that often turn out to be irrelevant. The goal of this project is to show how machine learning can complement current cybersecurity practices by automatically detecting anomalies in network traffic and classifying the associated threat — reducing the "human in the loop" workload for known attacks so analysts can focus on genuinely unknown ones.

## Architecture

The pipeline runs in two stages:

```
Network flow (36 features)
        │
   StandardScaler + sliding window (20 × 36)
        │
   LSTM Autoencoder  (encoder → latent 32 → decoder)
        │
   Reconstruction error > anomaly threshold?
        │
   ┌────┴─────────┐
  No              Yes
   │               │
 BENIGN     XGBoost (multiclass, balanced)
                   │
        DoS Hulk / Slowhttptest / slowloris
```

1. **LSTM Autoencoder (detection).** Trained exclusively on benign traffic, it learns to reconstruct normal network-flow sequences. Sequences it cannot reconstruct well — i.e. with a reconstruction error above a calibrated threshold — are flagged as anomalous. Being unsupervised, it can flag attacks of any kind, including unknown ones.
2. **XGBoost (classification).** A supervised multiclass classifier (`multi:softprob`) that operates on the flows marked as anomalous and assigns the specific attack type.

## Dataset

[CIC-IDS2017](https://www.unb.ca/cic/datasets/ids-2017.html) (MachineLearningCVE) — Canadian Institute for Cybersecurity, University of New Brunswick. Features generated by CICFlowMeter over real traffic captures.

- `dataset/Monday-WorkingHours.pcap_ISCX.csv` — benign traffic, used to train the LSTM Autoencoder.
- `dataset/Wednesday-workingHours.pcap_ISCX.csv` — mixed traffic with DoS Hulk, Slowhttptest and slowloris attacks, used to train and evaluate XGBoost.

> The CSV files are stored with [Git LFS](https://git-lfs.github.com/). Install it (`git lfs install`) before cloning to retrieve the actual data.

## Methodology

**Preprocessing**
- Cleaned NaN and ±inf values (a known CICFlowMeter extraction artifact).
- Reduced each dataset to 50,000 rows, preserving order (required by the LSTM).
- Feature selection via a correlation matrix (0.90 threshold) plus domain documentation, ending with 36 relevant features.
- `StandardScaler` fitted only on benign traffic to avoid data leakage.
- `LabelEncoder` for the attack labels.
- Sliding windows of 20 consecutive flows (stride 1) to build the 3D tensor the LSTM requires.

**Training**
- LSTM Autoencoder: unsupervised on benign Monday traffic, MAE loss, Adam optimizer, batch size 256, early stopping. ~55k trainable parameters, converged in ~48 epochs.
- Anomaly threshold set at the 95th percentile of the benign validation reconstruction error (final value: 0.4426).
- XGBoost: supervised on 70% of Wednesday traffic, stratified split, balanced sample weights, 30 estimators, max depth 6, learning rate 0.1.

## Results

**XGBoost in isolation** (test set, stratified): F1-score ≈ 1.00 across all four classes — confirming the selected features are sufficient when labels are available.

**Full pipeline (LSTM + XGBoost):**

| Class            | Precision | Recall | F1   | Support |
|------------------|-----------|--------|------|---------|
| BENIGN           | 0.72      | 1.00   | 0.84 | 25,466  |
| DoS Hulk         | 1.00      | 0.42   | 0.59 | 14,887  |
| DoS Slowhttptest | 1.00      | 0.90   | 0.95 | 5,499   |
| DoS slowloris    | 0.99      | 0.86   | 0.92 | 4,129   |
| **Accuracy**     |           |        | 0.80 | 49,981  |

- **Perfect recall on BENIGN (1.00):** legitimate traffic is never misclassified as an attack — no false alarms.
- **Near-perfect precision on attack types:** when the system names a threat, it is almost always right.
- **Low recall on DoS Hulk (0.42):** a limitation inherited from the detection stage — DoS Hulk generates seemingly valid HTTP requests whose features overlap with benign traffic, so its reconstruction error stays below the anomaly threshold and ~9,750 flows never reach XGBoost.

## Limitations & future work

- Attacks that do not noticeably perturb flow features (DoS Hulk) escape the unsupervised detector. Possible improvements: additional features that better characterize these flows, larger sliding windows (20 → 100), or combining the LSTM with another unsupervised detector such as Isolation Forest.
- The dataset was reduced for runtime; scaling to the remaining CIC-IDS2017 attack classes (PortScan, Brute Force, Web Attacks, Infiltration, Botnet) would test how the architecture generalizes.
- Adding an LLM-generated explanation step that takes the highest-error features, the predicted class and flow context to produce a human-readable report for analysts.
